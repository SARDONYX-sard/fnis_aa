# FNIS Alternate Animation — SKSE Plugin

SKSE plugin that replaces the FNIS alternate animation Papyrus runtime
(`FNIS_aa2.pex`) with a JSON-driven native implementation.
Existing scripts that call `FNIS_aa` functions work without modification.
No FNIS generator required.

## Requirements

- Skyrim Special Edition / VR
- [SKSE64](https://skse.silverlock.org/)
- [BehaviorDataInjector](https://www.nexusmods.com/skyrimspecialedition/mods/78146)
- [d_merge](https://github.com/SARDONYX-sard/d-merge) — generates `config.json` and BDI config

## Installation

1. Install via your mod manager.
2. Run **d_merge** to generate the following files:
   - `SKSE/Plugins/BehaviorDataInjector/FNIS_AA_to_OAR_BDI.json`
   - `SKSE/Plugins/fnis_aa/config.json`
3. Launch the game. No further setup needed.

## How It Works

`FNIS_aa2.pex` is normally binary-patched by the FNIS generator to embed the
slot layout. This plugin ships a native stub `FNIS_aa2.pex` that delegates all
calls to C++ implementations, reading the slot layout from `config.json`
produced by d_merge instead.

`FNIS_aa.pex` is left untouched — it internally calls `FNIS_aa2` functions
which are intercepted natively.

```txt
d_merge (patcher)
├── SKSE/Plugins/BehaviorDataInjector/FNIS_AA_to_OAR_BDI.json
│   └── BDI injects FNISaa_* variables into Havok behavior graphs
└── SKSE/Plugins/fnis_aa/config.json
    └── this plugin loads mod/group/base slot layout at startup
                     ↓
     Existing scripts call FNIS_aa functions as usual
                     ↓  (FNIS_aa2 natives intercepted by this plugin)
     actor.SetAnimationVariableInt("FNISaa_<group>", base + style)
```

### config.json

Generated by d_merge. Located at `SKSE/Plugins/fnis_aa/config.json`.

```json
{
  "crc": 123456,
  "fnis_version": "V07.06.00.0",
  "mods": [
    {
      "prefix": "xpe",
      "name": "XPMSE",
      "mod_id": 0,
      "groups": [
        { "name": "_1hmeqp", "base": 1 },
        { "name": "_dageqp", "base": 1 }
      ]
    },
    {
      "prefix": "xpe2",
      "name": "XPMSE",
      "mod_id": 0,
      "groups": [
        { "name": "_1hmeqp", "base": 9 },
        { "name": "_dageqp", "base": 2 }
      ]
    }
  ]
}
```

| Field          | Description                                                                    |
| -------------- | ------------------------------------------------------------------------------ |
| `crc`          | Fingerprint of the slot layout. Invalidates saved values on load order change. |
| `fnis_version` | FNIS version string returned by `FNISVersionGenerated.get()`.                  |
| `prefix`       | 3-character mod prefix used by FNIS (e.g. `xpe` for XPMSE).                    |
| `mod_id`       | Index in load order. Assigned by d_merge.                                      |
| `base`         | First slot index occupied by this mod for this group.                          |

> `group_id` are omitted from the JSON — both are resolved
> statically at runtime from the built-in group table (`kAltGroupTable`).

- The next mod base value increases by the number of registered animations.

- Bases with the same group_name must not be duplicated.

## Overridden Papyrus Functions

The following functions in the `FNIS_aa2` class are replaced by native
implementations. All other `FNIS_aa` / `FNIS_aa2` functions remain as-is.

```papyrus
; Returns mod count, set count, or crc depending on listType (0, 1, 2).
int Function GetAAnumber(int listType) global native

; Returns the list of 3-character mod prefixes indexed by mod_id.
string[] Function GetAAprefixList(int nMods, string mod, bool debugOutput = false) global native

; Returns the list of encoded set strings "PPGGBB" for all registered groups.
string[] Function GetAAsetList(int nSets, string mod, bool debugOutput = false) global native
```

## Slot Layout

Slot 0 is always reserved for vanilla. Custom slots start at 1 and are
assigned contiguously per group in mod load order.

```txt
slot 0        vanilla
slot 1..N     mod A  (base=1, slot_count=N)
slot N+1..M   mod B  (base=N+1, slot_count=M-N)
```

`slot_count` per group is fixed by the FNIS alternate animation spec and is
resolved statically from the built-in `kAltGroupTable` at runtime — it is not
stored in `config.json`.

## CRC and Save Compatibility

The `crc` field is a fingerprint of the current slot layout. On each game
load, the plugin writes the current CRC to `FNISaa_crc` on the actor.
If the value differs from what was saved, the calling Papyrus script should
re-apply all animation variables to reflect the updated layout.

## Uninstallation

1. Reset all alternate animations to vanilla before uninstalling
   (call `FNIS_aa.SetAnimGroupEX(akActor, groupName, 0, 0, "XPMSE")` for each group).
2. Remove the plugin and generated JSON files.
